3. Дата и время. 1) Абсолютные 2) Относительные (интервальные)
4. Булевский тип (TRUE, FALSE, UNKNOWN). Операции (NOT, AND, OR)
Домен - понятие, основанное на типе данных. Можно представлять домен как пользовательский тип данных. 

Основные понятия реляционной модели данных.
Реляционная теория - Язык SQL
1. 	Отношение (Relation) - Таблицы (Table)
	r - отношение
	Hr (заголовок) - множество упорядоченных { <A,T> }
2.	Атрибут (Atribute) - Столбец (Column). Каждый атрибут в заголовке отношения должны быть различны
3.	Кортеж (Tuple) - Строка или запись (Row). 
	tr { <A,T,v> }
4.	Тело отношения - Содержимое (множество строк) таблицы
	Br - произвольное множество кортежей, принадлежащих отношению tr
	Содержание таблицы может меняться
5.	Заголовок (схема) отношения - Схема таблицы
	Vr = <Hr,Br>
6. 	Переменное отношение - Именованная таблица БД во времени
	VAR r - именованный контейнер, который может содержать 
Любую операцию над БД можно представить в виду VAR r := <Hr, Br>
Степенью отношения обычно называют количество атрибутов в заголовке отношения
Схемой отношения называется множество { <VAR r, Hr> }
Базой данных называется множество { <VAR r, Vr> }
Изначально в реляционной теории считалось, что сначала определяется схема базы данных, а потом схема базы данных не меняется, а меняются только тела отношений. Однако из практических соображений выяснилось, что такой подход не очень удобен. 

Первичный ключ отношения - такое подмножество заголовка, которое обладает двумя свойствами: 1) Уникальность - в любой момент времени значение первичного ключа в любом кортеже тела отношения отличается от значения первичного ключа в любом другом кортеже в этом же теле отношения; 2) Минимальность - никакое собственное подмножество этого множества атрибутов свойством уникальности не обладает. Первичный ключ в каждом отношении может быть только один. Если же существует несколько наборов атрибутов, обладающих уникальностью и минимальностью, то только один должен быть назначен первичным ключом. Выбор первичного ключа - выбор администатора базы данных. Первичный ключ является ограничением целостности отношения и при выполнении операций выполяется проверка. Атрибуты первичного ключа не должны содержать неопределенных или NULL значений
VAR r
S in Hr

Фундаментальные свойства отношений
1. Отсутствие кортежей-дубликатов
2. Наличие первичного ключа
3. Тело отношения определяется как множество кортежей, а в теории множеств все элементы множества по определению различны
4. Из свойства отсутствия кортежей-дубликатов вытекает свойство наличия первичного ключа
5. Свойством всегда обладает по крайней мере полный набор атрибутов отношения. Если в результате удаления какого-то атрибута уникальность пропадает, то его необходимо вернуть в состав первичного ключа. Если остается, то этот атрибут можно удалить из первичного ключа. В разном порядке первичный ключ может получится разным. Разные результаты могут получится, если есть несколько вариантов первичного ключа.
В практических реализациях наличие первичного ключа является необязательным, а всего лишь желательным. В реляционной модели всегда нет дубликатов.
